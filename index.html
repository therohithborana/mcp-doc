What is Anarix MCP?

Anarix MCP is a Model Context Protocol (MCP) server that connects Claude with your Anarix account. It lets you query advertising and marketplace data conversationally — simply describe what you want to know and Anarix MCP fetches the relevant data, runs analysis, and returns results directly inside Claude.

Anarix MCP is built for real operational analysis. It connects to live account data, not a static dataset — so every response reflects current performance.

Installation

Installation & Setup

Setting up Anarix MCP takes about 2 minutes. You can do this from the Claude app or directly in the Claude website using Custom Connectors.

1
Open Claude

Open either the Claude desktop app or the Claude website in your browser and sign in to your account.

2
Open Settings

Click Settings, then go to Connectors and click Customize.

3
Add a Custom Connector
  1. Click Add Connector
  2. Click Add Custom Connector
  3. Set the name to JIVA
  4. Set the remote MCP server URL to https://mcp.anarix.ai/mcp/sse
  5. Click Add

Once you click Add, the JIVA MCP connector will be available in Claude.

4
Verify the JIVA Connection

After adding the connector:

  1. Open a new chat
  2. Look for the tools / connectors icon (hammer icon or plug icon)
  3. Confirm JIVA appears in the list
💡
Tools not showing?

Refresh Claude or reopen the app/site, then check the connectors list again.

Authentication

Authentication Flow

Anarix MCP uses a secure browser-based login flow. You'll only need to do this once per session or when your session expires.

1
Trigger Login

Start a new chat and type any JIVA-related query, such as:

use Jiva and show me the accounts I have access to.

Anarix MCP will detect that you are not authenticated and generate a login URL.

2
Open the Login URL

Click the URL provided by Anarix MCP. It will open a browser window where you can sign in with your Anarix credentials.

3
Complete Sign-In

Enter your Anarix username and password and complete the sign-in. Return to Claude after that.

⚠️
Don't delay

The login link is time-limited. Complete the browser sign-in promptly and return to Claude to avoid session expiry.

4
Session Established

Anarix MCP will confirm authentication and list the accounts available to you. You're now ready to start querying.

Re-authentication

You may need to log in again when:

  • Starting a new Claude session after a long gap
  • Your session token has expired
  • You switch accounts or clear local data

Simply trigger a new login query and repeat the flow above.

Quick Start

Your First Test

Once authenticated, start with these prompts to confirm everything is working:

Step 1 — Check account access

Show me the accounts I have access to.

Step 2 — Run a performance query

What were my top 5 campaigns by ad spend from 2026-04-01 to 2026-04-30?

Step 3 — Request a visualization

Create a chart showing weekly ad spend and ROAS for the last 8 weeks.
✅
All three worked?

You're fully set up. Anarix MCP is connected, authenticated, and pulling live data from your Anarix account.

Support

Troubleshooting

If Anarix MCP does not appear or authenticate correctly, use the checks below before escalating to your Anarix admin.

I can't find the custom connector option
  • Open Claude Settings and look for Connectors.
  • Click Customize to manage custom connectors.
  • If you still do not see it, refresh the browser or reopen the Claude app.
JIVA does not appear in the tools or connectors list
  • Go back to Connectors and confirm the custom connector was added with the name JIVA.
  • Make sure the remote MCP server URL is exactly https://mcp.anarix.ai/mcp/sse.
  • Refresh Claude or reopen the app/site and check again.
The connector was added, but it won't connect
  • Check that the remote MCP server URL is exactly https://mcp.anarix.ai/mcp/sse.
  • Remove the connector and add it again if the URL or name was entered incorrectly.
  • Refresh Claude and retry from a new chat.
Login keeps repeating or the sign-in link expires
  • Trigger a fresh Anarix MCP query to generate a new login URL.
  • Complete the browser login immediately after opening the link.
  • You may need to sign in again after a long gap, after clearing local data, or when the session token expires.
I can log in, but I don't see the expected accounts
  • Ask Anarix MCP: "Show me the accounts I have access to."
  • Anarix MCP only exposes accounts your Anarix credentials are authorized to access.
  • If an expected account is missing, contact your Anarix admin to confirm permissions.
